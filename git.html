GIT WINDOWS

Conceito O git mapeia os arquivos da pasta do projeto e sabe quais arquivos sofreram alterações bem como as linhas que sofreram alterações. Com isso ele cria uma lista e no momento de atualizar o arquivo (no ambiente de desenvolvimento - em nuvem) ele altera só o que foi alterado. Sem necessidade de carregar todo arquivo novamente. Trabalha com conceito de "Branches" (varias versões do mesmo código) trabalhando em paralelo.
Passo a passo
  1. Criar conta em Git Hub
  2. Baixar e instalar Git SCM
  3. Escolher uma pasta de projeto ou criar uma

*Na pasta do projeto executar os comandos

  • Git init - cria o git na pasta
  • LS -a (enxerga diretorio)
  • Mk Dir - cria diretório
  • CD - entra na pasta
Operações com git
  1. git status - mostra status dos arquivos da contexto
  2. git update - atualiza
  3. git revert - reverte
  4. git add - adiciona arquivos ao git
  5. git add untracked -
  6. git patch - atualiza arquivos
  7. git diff - mostra diferenças entre versões de um mesmo arquivo
  8. git quit - sai do git
  9. git rm - remove arquivo da lista e da pasta do projeto
  10. git commit - grava alterações incluídas
