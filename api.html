API Acrônimo - Application Programming Interface - Interface de aplicação de programação.

Analogia:

  1. Garçon - Navegador
  2. Cliente - Cliente
  3. Cozinha - Servidor

O garçon anota o pedido (chrome), e leva até a cozinha (envia pela internet - usando REST (via http) - até o servidor.)

Torna um sistema web mais escalável e mais abrangente pois uma API pode ser consumida por qualquer plataforma, seja um cliente pc (navegador um app desktop), ou mobile (navedor ou app-mobile).

REST Acrônimo: Representational State Transfer - É basicamente um cumprimento de regras na troca de mensagens entre servidor e cliente. Determina algumas obrigações (constraints) dentro de uma transferência, usando HTTP, estabelece regras.
As 6 regras abaixo , quando cumpridas qualificam uma API REST como Restfull.
  1. Cliente Servidor
    2. *Implicações: Portabilidade do sistema.
  2. _Stateless_ : o pedido deve ser completo, protocolo com todas informações, não somente o pedido. Entre elas a sessão do usuário.
    Analogia: Como uma senha do banco.
  3. Cacheable: As respostas devem explicitar se podem ser cachedas.
  4. Layered System: Sistema por camadas
    Criação de endpoints.
  5. Code on demand: quando o servidor envia scripts para serem executados no cliente.
  6. Uniform Interface: manter o padrão na aplição quanto ao formato usado para troca de mensagens (json, xml, etc)
    assim como no uso dos "verbos" HTTP como (PUT, GET, POST, DELETE, PATCH)
TIPOS DE API
  1. Privada - internamente dentro do servidor
  2. Parceiros - entre parceiros (mais provável B2B).
  3. Pública - Podem ser utilizadas livremente.
